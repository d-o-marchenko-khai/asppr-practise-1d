Короткий опис розробленої програми

Програма реалізована мовою C# на платформі .NET 8 у вигляді консольного застосунку. Вона розв'язує задачу цілочислового лінійного програмування методом Гоморі. Алгоритм складається з трьох фаз: пошук опорного (допустимого) розв'язку, пошук оптимального розв'язку без вимоги цілочисловості та ітеративне відсікання дробових розв'язків додатковими обмеженнями s_k за методом Гоморі. На кожній ітерації програма виводить симплекс-таблицю з позначками змінних, рядок і стовпець розв'язувального елемента, перераховані значення та сформоване додаткове обмеження.

Структура програми

Програма розбита на 13 класів.

Program - точка входу. Містить головне меню з п'ятьма пунктами (пошук опорного розв'язку, пошук оптимального розв'язку, максимум/мінімум функції мети Z, пошук цілочислового розв'язку методом Гоморі, тестові обчислення), яке з'являється повторно після кожної операції.

LpProblemInput - зчитування постановки задачі з консолі. Запитує кількість змінних і обмежень, напрямок оптимізації (max/min), коефіцієнти функції мети, кожне обмеження у форматі "a1 a2 ... an  знак  b" та номери цілочислових змінних (порожньо, перелік через пробіл або ключове слово all).

TestRunner - автоматичне виконання тестових прикладів на захардкожених даних без участі користувача: варіант завдання та два приклади з методички.

JordanElimination - процедура МЖВ. Розв'язувальний елемент замінюється на 1/a_rs, елементи рядка діляться на нього, елементи стовпця діляться зі зміною знаку, решта перераховується за формулою b_ij = a_ij - a_is * a_rj / a_rs.

SimplexTable - структура симплекс-таблиці. Зберігає саму матрицю, позначки стовпців (-x1, -x2, ...) і рядків (y1, y2, ...). Містить методи SwapLabels (міняє місцями позначки розв'язувального рядка та стовпця після МЖВ), RemoveRow, RemoveColumn та AddRow для динамічної зміни розміру таблиці при додаванні відсікаючих обмежень Гоморі.

ProblemBuilder - приводить усі обмеження до вигляду "... >= 0" (нерівності <= домножаються на -1) і будує вхідну симплекс-таблицю. Виводить переписану систему обмежень у протокол.

NullRowEliminator - видаляє з таблиці нуль-рядки, що відповідають обмеженням-рівностям, послідовним застосуванням МЖВ із викресленням відповідного 0-стовпця; при ненульовому вільному члені у виродженому рядку фіксує суперечливість системи.

BasicSolutionSolver - пошук опорного розв'язку. Шукає від'ємний елемент у стовпці вільних членів, у його рядку - від'ємний елемент, це розв'язувальний стовпець. Далі за мінімальним невід'ємним відношенням обирає розв'язувальний рядок і викликає МЖВ. Завершується, коли всі вільні члени невід'ємні, або повідомляє про суперечливість системи.

OptimalSolutionSolver - пошук оптимального розв'язку для задачі максимізації. Шукає від'ємний елемент у Z-рядку - це розв'язувальний стовпець. Розв'язувальний рядок обирається за мінімальним невід'ємним відношенням b_i / a_ij серед додатних елементів стовпця. Завершується, коли Z-рядок не містить від'ємних елементів, або повідомляє про необмеженість функції мети.

GomorySolver - алгоритм Гоморі для цілочислового програмування. Серед базисних змінних, що мають вимогу цілочисловості, обирає ту, у якої вільний член має максимальну дробову частину; за її рядком будує додаткове обмеження s_k = sum(B_ij * E_j) - B_i >= 0, де B_ij та B_i - дробові частини відповідних коефіцієнтів. Нове обмеження додається в таблицю окремим рядком (із від'ємним вільним членом), після чого повторно виконуються пошук опорного й оптимального розв'язків. Цикл повторюється, доки всі цілочислові змінні не отримають цілих значень або не буде виявлено суперечливість/необмеженість (з обмеженням на кількість ітерацій).

MinToMaxConverter - перетворення задачі мінімізації на задачу максимізації. Просто змінює знаки коефіцієнтів функції мети: Z' = -Z. Після розв'язання максимум Z' дорівнює мінус мінімуму Z.

LinearProgrammingSolver - оркестратор. Виводить постановку задачі (включно з переліком цілочислових змінних), при потребі перетворює min на max, будує таблицю, викликає фази пошуку та витягає значення змінних з базису. Залежно від режиму SolveMode зупиняється після опорного розв'язку, виводить оптимальний розв'язок, друкує фінальне Max(Z)/Min(Z) або додатково запускає GomorySolver і виводить цілочисловий оптимальний розв'язок.

Protocol - форматований вивід протоколу. Друкує симплекс-таблицю з вирівняними колонками, позначки розв'язувальних рядка та стовпця, опорний та оптимальний розв'язки, додаткові обмеження s_k й підсумкове значення функції мети.

Вхідні та вихідні дані

Для пунктів 1-4 меню користувач вводить кількість змінних n та кількість обмежень m, напрямок оптимізації (max або min), n коефіцієнтів функції мети через пробіл, m обмежень у форматі "a1 a2 ... an  знак  b" (де знак - це <=, >= або =) та перелік номерів цілочислових змінних.

Пункт 1 виводить протокол пошуку опорного розв'язку та сам опорний розв'язок X = (x1, ..., xn). Пункт 2 додатково виконує фазу пошуку оптимуму і виводить оптимальний розв'язок. Пункт 3 до всього цього додає фінальне значення Max(Z) або Min(Z). Пункт 4 додатково виконує метод Гоморі та виводить цілочисловий оптимальний розв'язок із відповідним значенням Max(Z)/Min(Z).

Пункт меню "Тестові обчислення" виконує повне розв'язання на трьох заздалегідь підготовлених задачах: варіант завдання Z = x1 + x3 + x6 -> max (всі змінні цілі) та два приклади з методички - Z = x1 + 4x2 -> max з очікуваним цілочисловим розв'язком X = (1; 1), Max(Z) = 5 та Z = 4x1 + 5x2 + x3 -> max з очікуваним цілочисловим розв'язком X = (2; 2; 1), Max(Z) = 19.

Елементи рядка вводяться через пробіл. Числа приймаються як через крапку, так і через кому. Протокол обчислень виводиться безпосередньо в консоль.
